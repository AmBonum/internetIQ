# E8.2 — `/test/zostav` composer UI

**Epic:** [E8 — Composer](../PLAN-2026-04-26-custom-tests-sponsorship.md#epic-8--composer-firmy-zostavujú-vlastné-testy)
**Effort:** `L` (≤ 4 dni)
**Priority:** `P2` (should)
**Status:** ⛔ Blocked by E8.1, E7.1

## Kontext

Composer je hlavný **B2B nástroj** projektu — firma si vyskladá test
na mieru pre svoj tím. UX musí byť dosť silné aby technologicky
zdatný majiteľ to spravil za 90 sekúnd, ale aj dostatočne jasné aby
laik to zvládol.

Zjednocuje dva pôvodne oddelené flow (ad-hoc picker a industry packy)
do jedného: pack-y sa stávajú **sugesciami / preset-mi**, nie striktne
oddelenými cestami.

## User story

> Ako majiteľ firmy, **chcem** v jednom rozhraní vybrať jeden alebo viac
> industry packov, doplniť/odobrať konkrétne otázky a nastaviť vlastný
> prah úspešnosti pre môj tím, **aby** výsledný test odrážal presne to
> čo potrebujem ja, nie čo predpokladá content autor.

## Acceptance criteria

- **AC-1**: Route `/test/zostav` (file-based: `src/routes/test.zostav.tsx`).
- **AC-2**: **Step 1 — Pre-load packov** (voliteľné):
  - Multi-select chips so všetkými industry packmi (`getPacksByIndustry()`)
  - Klik na chip → otázky packu sa pridajú do výberu (deduplicated cez `Set`)
  - Klik znova → otázky packu sa odstránia
  - Tooltip / sub-text: „Pridáva ~12 otázok pre {industry}"
- **AC-3**: **Step 2 — Filter & picker**:
  - Filter chips kategórie (`phishing`, `url`, `fake_vs_real`, `scenario`, `honeypot`) — multi-select
  - Filter chips obtiažnosti (`easy`, `medium`, `hard`) — multi-select
  - Search input (case-insensitive substring v `prompt`)
  - Question list: každá s checkboxom + krátky preview (prompt + category + difficulty)
  - Counter: „Vybraných: 17 / 50"
  - Disable check ak už 50 vybraných
- **AC-4**: **Step 3 — Settings**:
  - Slider „Maximálne otázok" (5–50, default 15) — orezáva výber ak prekročené
  - Slider „Prah úspešnosti" (50–90, default 70) — sub-text s vysvetlením („Skóre ≥ 70 → zamestnanec dostane badge ‚Vyhovuje pre {label}'")
  - Voliteľný input „Pomenovanie zostavy" (max 80 znakov, napr. „E-shop Q1 2026 onboarding")
  - Live preview honeypot ratio v zostave („Aktuálne 22 % legit otázok")
- **AC-5**: **Step 4 — Akcie** (sticky bottom bar):
  - „Zdieľať zostavu" — POST do `/api/test-sets`, redirect `/test/zostava/{id}`
  - „Spustiť pre seba" — bez DB save, redirect `/test/zostav?config=base64...` (≤10 otázok) alebo presmerovanie cez DB pri väčších zostavách
  - „Vyčistiť výber" — confirm dialog ak už ≥10 otázok
- **AC-6**: Min 5 otázok aby tlačidlá boli aktívne. Vizuálny error state ak `<5`.
- **AC-7**: Mobile-first: kroky sa stack-ujú vertikálne, sliders dotyk-friendly, filter chips wrapuju, picker list scroll-snap.
- **AC-8**: A11y:
  - Každá sekcia má `<section aria-labelledby>`
  - Sliders s `aria-valuemin/max/now`
  - Filter chips `aria-pressed`
  - Counter v `aria-live="polite"` regióne
  - Keyboard nav cez celé UI (Tab + Space)
- **AC-9**: SEO: `<title>Zostav vlastný test pre tím — Internet IQ Test</title>`, robots index follow, canonical.
- **AC-10**: Performance: 200+ otázok renderuje plynulo cez virtualizovaný list (react-window alebo simple intersection-observer-based).
- **AC-11**: B2B copy: „Pre tvoj tím", „Pre zamestnancov" — tone hovorí B2B, nie B2C.
- **AC-12**: URL share fallback: `?config=base64({ ids, threshold, max })` pre ≤10 otázok bez DB save.
- **AC-13**: **Stale question_id detection** — pri pre-load packu cez `getPackBySlug(slug)`, composer prejde každý ID cez `getQuestionById()`; ak count nesúhlasí (po refaktore questions.ts mohol ID zmiznúť), zobrazí toast warning „Z packu {label} chýba {n} otázok, autor packu ich pravdepodobne premenoval. Načítané iba {m} platných." Filter sa neapplikuje silently. Edge-case je rare ale realistický pri budúcich content refactor-och.

## Riziká

| Riziko | Mitigácia |
|---|---|
| 200+ otázok v DOM — performance | Virtualizovaný list, paginácia 50 per scroll |
| Filter UI prepchaný na mobile | Collapse na „Filtre (3)" sheet/drawer pri viewport <768px |
| Pack pre-load + ručné edit-y → user stratí prehľad | Vizuálny diff: „Z packu E-shop sa pridalo 12 otázok, ručne pridané 3" |
| Threshold slider pri prvom použití nezrozumiteľný | Live preview: „Pri skóre 75/100 → vidí badge" — zobrazené pri každom posune |

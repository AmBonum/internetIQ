# E8.2 — `/test/builder` filter + picker + start

**Epic:** [E8 — Manual test builder](../PLAN-2026-04-26-custom-tests-sponsorship.md#epic-8--manual-test-builder)
**Effort:** `M` (≤ 2 dni)
**Priority:** `P2` (should)
**Status:** ⛔ Blocked by E8.1

## Kontext

UI pre user-driven výber otázok. Ľudia chcú browsovať bank, filtrovať
podľa kategórie / obtiažnosti / textu, vybrať 5–25 otázok a spustiť
test (alebo zdieľať).

## User story

> Ako pokročilý user, **chcem** si zostaviť test z konkrétnych otázok,
> **aby** som mohol pripraviť cielený training pre kolegu / člena rodiny.

## Acceptance criteria

- **AC-1**: Route `/test/builder`:
  - Filter chips: kategória (`phishing`, `url`, …) — multi-select
  - Filter chips: obtiažnosť (`easy`, `medium`, `hard`)
  - Search input (case-insensitive substring v `prompt`)
- **AC-2**: Question list:
  - Každá otázka má checkbox + krátky preview (prompt + category + difficulty)
  - Counter: „Vybraných: 7 / 25"
  - Disable check ak už 25 vybraných
- **AC-3**: Bottom sticky bar:
  - „Spustiť test" — okamžite spustí test bez DB save (URL: `/test/builder?ids=...` — base64 ak ≤ 10 IDs, inak tlačidlo neaktivne kým user nezdieľa)
  - „Zdieľať zostavu" — INSERT do `test_sets`, redirect na `/test/zostava/{id}` (E8.3)
  - „Vyčistiť výber"
- **AC-4**: Min 5 otázok aby tlačidlo „Spustiť" bolo aktívne.
- **AC-5**: Mobile-first: chips wrap, list scroll-snap, bottom bar fixed.
- **AC-6**: A11y: keyboard nav (Tab medzi chips, Space toggle), `aria-pressed` na chips, `aria-checked` na otázkach, screen reader announce-uje počet vybraných pri každej zmene.
- **AC-7**: SEO: `<title>Vyskladaj si vlastný test — Internet IQ Test</title>`, robots index follow.

# E8.1 — `test_sets` table migration + RLS + retention

**Epic:** [E8 — Manual test builder](../PLAN-2026-04-26-custom-tests-sponsorship.md#epic-8--manual-test-builder)
**Effort:** `XS` (< 2h)
**Priority:** `P2` (should)
**Status:** 🟡 Ready

## Kontext

Manuálne zostavený test treba uložiť do DB iba ak ho user zdieľa
linkou. Tabuľka je jednoduchá: ID + question IDs JSONB + created_at.

## User story

> Ako user, ktorý si vyskladal vlastný test, **chcem** ho zdieľať
> stabilnou linkou, **aby** ho mohol kolega otvoriť bez registrácie.

## Acceptance criteria

- **AC-1**: Migrácia `supabase/migrations/{timestamp}_test_sets.sql`:
  ```sql
  CREATE TABLE public.test_sets (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    question_ids TEXT[] NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    creator_label TEXT,
    CONSTRAINT test_sets_size_chk CHECK (array_length(question_ids, 1) BETWEEN 5 AND 25)
  );
  ALTER TABLE public.test_sets ENABLE ROW LEVEL SECURITY;
  ```
- **AC-2**: RLS:
  - anon `SELECT` povolený na všetky riadky (test sety sú „verejné" linkou)
  - anon `INSERT` povolený s validáciou cez CHECK constraint
  - žiadny UPDATE/DELETE pre anon
- **AC-3**: Retencia: pridať `test_sets` do `purge_expired_attempts()` SQL funkcie alebo vytvoriť samostatnú `purge_unused_test_sets()` ktorá maže `WHERE created_at < now() - interval '12 months' AND NOT EXISTS (...)`. Default 12 mesiacov.
- **AC-4**: Migrácia pridaná aj do `DEPLOY_SETUP.sql`.
- **AC-5**: Supabase types regenerované cez `npx supabase gen types typescript`.

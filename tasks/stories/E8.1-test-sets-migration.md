## Kontext

Composer-zostavený test ukladáme do DB až keď ho firma zdieľa.
Schéma musí podporovať: zoznam otázok, custom passing threshold,
max otázok, source pack slugs, voliteľné pomenovanie.

## User story

> Ako firma, ktorá si vyskladala vlastný test pre zamestnancov, **chcem**
> ho zdieľať stabilnou linkou s vlastným prahom úspešnosti, **aby** sa
> zamestnanci nemuseli registrovať a zostava odrážala naše požiadavky.

## Acceptance criteria

- **AC-1**: Migrácia `supabase/migrations/{timestamp}_test_sets.sql`:
  ```sql
  CREATE TABLE public.test_sets (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    question_ids TEXT[] NOT NULL,
    passing_threshold INT2 NOT NULL DEFAULT 70,
    max_questions INT2 NOT NULL,            -- redundantné voči array_length, ale rýchlejšie pre UI
    creator_label TEXT,                      -- voliteľný popis: napr. "E-shop Q1 2026 onboarding"
    source_pack_slugs TEXT[],                -- audit: z ktorých industry packov sa odvodzuje
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    CONSTRAINT test_sets_size_chk CHECK (
      array_length(question_ids, 1) BETWEEN 5 AND 50
    ),
    CONSTRAINT test_sets_threshold_chk CHECK (
      passing_threshold BETWEEN 50 AND 90
    ),
    CONSTRAINT test_sets_max_consistent CHECK (
      max_questions = array_length(question_ids, 1)
    ),
    CONSTRAINT test_sets_label_len CHECK (
      creator_label IS NULL OR length(creator_label) <= 80
    ),
    CONSTRAINT test_sets_question_id_len CHECK (
      NOT EXISTS (SELECT 1 FROM unnest(question_ids) AS q WHERE length(q) > 64)
    )
  );
  ALTER TABLE public.test_sets ENABLE ROW LEVEL SECURITY;
  CREATE INDEX test_sets_created_at_idx ON public.test_sets (created_at DESC);

  -- Link attempts back to the test_set that generated them, for analytics.
  ALTER TABLE public.attempts
    ADD COLUMN test_set_id UUID NULL REFERENCES public.test_sets(id) ON DELETE SET NULL;
  ```
- **AC-2**: RLS:
  - anon `SELECT` povolený na všetky riadky (composeri sú „verejné" linkou)
  - anon `INSERT` povolený s validáciou cez CHECK constraints
  - žiadny UPDATE/DELETE pre anon
- **AC-3**: Retencia: samostatná `purge_unused_test_sets()` ktorá maže
  `WHERE created_at < now() - interval '12 months'`. Naplánovaná cez
  pg_cron rovnako ako `purge_expired_attempts`.
- **AC-4**: Migrácia pridaná aj do `DEPLOY_SETUP.sql`.
- **AC-5**: Supabase types regenerované cez `npx supabase gen types typescript`.
- **AC-6**: Server-side validácia (v Supabase RPC alebo CF Pages Function pre INSERT) overuje, že každý `question_ids[*]` existuje v aktuálnom `QUESTIONS` poolu — chráni proti orphan IDs po refaktore questions.ts. Reject 400 ak found unknown ID.
- **AC-7**: Cloudflare Pages Functions rate limit na POST endpoint: max **5 requests / minute / IP**, edge-level cez CF native rate limiting. Bráni spam-create test_sets.
- **AC-8**: `attempts` migration ALTER pridáva `test_set_id` FK column. `forbid_attempt_score_changes` trigger update — pridať `test_set_id` ako immutable po INSERT (nesmie sa meniť).
- **AC-9**: `purge_unused_test_sets()` rešpektuje FK — najprv NULL-ifikuje `attempts.test_set_id` (ON DELETE SET NULL), potom DELETE. Migrácia testovaná lokálne s mock dátami.

# AH-11.1 — Replace mock-store with React Query hooks for ADMIN domain

**Epic:** [AH-11 — Mock → Supabase wiring + production-ready verification](../PLAN-2026-05-17-admin-hub-integration.md)
**Effort:** `L` (≤ 1 week)
**Priority:** `P1` (must)
**Status:** Backlog

## Goal

Swap every `from "@/lib/admin/mock-store"` import across the admin domain
(questions, answer-sets, tests, categories, trainings, respondents, audit,
DSR, reports, users) for TanStack Query hooks backed by the real Supabase
anon client. UI is byte-identical; only the data source changes.

## Acceptance criteria

- All ported files have `npx eslint --fix` applied; `npm run lint` is 0/0.
- `grep -r "from \"@/lib/admin/mock-store\"" src/` returns zero matches.
- `grep -r "from \"@/lib/admin/mock-data\"" src/` returns zero matches.
- No new mock files created; mock files are DELETED in AH-11.6.
- `data-testid` values preserved from AH-3..AH-10; no new test-ids added.
- i18n keys unchanged; no new namespaces.
- Every list/detail screen uses `useQuery` with stable `queryKey`; every
  mutation uses `useMutation` + `qc.invalidateQueries`.
- Loading / error / empty states match the existing visual treatment
  the mock store already exercises.
- `FEATURE_MAP-admin-hub.md` rows for each swapped component marked Done
  in the same commit.
- CONSENT_VERSION unchanged (1.4.0 from AH-7.1).

## Implementation

- Hooks live at `src/lib/admin/queries.ts` (queries + mutations grouped by
  domain — `useAdminQuestions`, `useAdminAnswerSets`, etc.).
- Edit every `src/routes/admin/**/*.tsx` and `src/components/admin/**/*.tsx`
  that imports `mock-store`; swap import to the hook module.
- Use the anon Supabase client (`@/integrations/supabase/client`) for reads
  — privileged writes are delegated to AH-11.3 server fns.
- `qc.invalidateQueries({ queryKey: ["admin", <domain>] })` after every
  mutation; no manual cache surgery.

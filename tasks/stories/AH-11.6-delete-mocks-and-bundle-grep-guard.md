# AH-11.6 — Delete mock store files + add `npm run audit:bundle` grep guard

**Epic:** [AH-11 — Mock → Supabase wiring + production-ready verification](../PLAN-2026-05-17-admin-hub-integration.md)
**Effort:** `S` (≤ ½ day)
**Priority:** `P0` (production-ready gate)
**Status:** Backlog

## Goal

Remove every mock store file ported in AH-3..AH-10 once AH-11.1..AH-11.5
have replaced their imports with Supabase-backed React Query hooks, and
add a CI grep guard that fails any future PR which leaks a mock path
into the production bundle.

## Acceptance criteria

- All six mock files deleted from `src/`:
  `src/lib/admin/mock-store.ts`, `src/lib/admin/mock-data.ts`,
  `src/lib/admin/answer-sets-mock-store.ts`,
  `src/lib/admin/cms-mock-store.ts`,
  `src/lib/platform/mock-store.ts`,
  `src/lib/platform/mock-user-data.ts`.
- `grep -r "mock-store\|mock-data\|mock-user-data" src/` returns zero matches.
- `npm run build` succeeds.
- `npm run audit:bundle` runs after `npm run build` and exits 0; greps
  `dist/` for `mock-store|mock-data|mock-user-data|admin-mock-data|platform/store|platform/seed`
  and fails with non-zero on any match (sourcemaps excluded via `grep -v "\.map$"`).
- `package.json` has a new `"audit:bundle": "bash scripts/audit-bundle.sh"` script.
- `scripts/audit-bundle.sh` is executable and ends with a clear PASS/FAIL line.
- CI workflow in `.github/workflows/*.yml` runs `npm run build && npm run audit:bundle`
  on every PR; the audit step blocks merge on failure.
- `npm run lint` is 0/0; `npm test` is green; `npm run e2e:browser` and
  `npm run e2e:integration` are green (all data-testids preserved, no UI changed).
- `FEATURE_MAP-admin-hub.md` rows for the deleted mock files marked Done.

## Implementation

- Delete the six mock files (single commit).
- Create `scripts/audit-bundle.sh`:
  ```sh
  #!/usr/bin/env bash
  set -euo pipefail
  if grep -rE "mock-store|mock-data|mock-user-data|admin-mock-data|platform/store|platform/seed" dist/ | grep -v "\.map$"; then
    echo "FAIL: mock leaked into prod bundle"
    exit 1
  fi
  echo "PASS: no mock in bundle"
  ```
- `chmod +x scripts/audit-bundle.sh`.
- Add `"audit:bundle"` script to `package.json`.
- Edit `.github/workflows/<existing-ci>.yml` (locate via `ls .github/workflows/`)
  to run `npm run build && npm run audit:bundle` on PR.
- Update `FEATURE_MAP-admin-hub.md` status column for the mock file rows.

## Tests

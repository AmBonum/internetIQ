# E10.4 — CSP + `.dev.vars` + Cloudflare env vars + Stripe keys

**Epic:** [E10 — Sponsorship infrastructure](../PLAN-2026-04-26-custom-tests-sponsorship.md#epic-10--sponsorship-infrastructure)
**Effort:** `XS` (< 2h)
**Priority:** `P1` (must)
**Status:** ⛔ Blocked by E10.1

## Kontext

Security boilerplate pred prvým kódovým commitom v sponsorship trace —
secrety, CSP rozšírenie, environment matrix.

## User story

> Ako vývojár / DevOps, **chcem** mať jasne definovaný set env vars a
> CSP zmenu pred prvým deploymentom Stripe webhooku.

## Acceptance criteria

- **AC-1**: `public/_headers` aktualizovaný:
  - `script-src 'self' 'unsafe-inline' https://js.stripe.com`
  - `connect-src 'self' https://*.supabase.co https://api.stripe.com https://q.stripe.com`
  - `frame-src 'self' https://js.stripe.com https://hooks.stripe.com`
  - `img-src 'self' data: https://*.stripe.com`
- **AC-2**: `.dev.vars` (in `.gitignore`) obsahuje:
  ```
  STRIPE_SECRET_KEY=sk_test_...
  STRIPE_WEBHOOK_SECRET=whsec_...
  STRIPE_PUBLISHABLE_KEY=pk_test_...
  SUPABASE_SERVICE_ROLE_KEY=...
  ```
- **AC-3**: `.dev.vars.example` (committed) ako šablóna bez real values.
- **AC-4**: Cloudflare Pages — env vars v dashboard pre **Production**:
  - `STRIPE_SECRET_KEY` (live), `STRIPE_WEBHOOK_SECRET` (live)
  - `STRIPE_PUBLISHABLE_KEY` (`VITE_` prefix pre frontend visibility, live)
  - `SUPABASE_SERVICE_ROLE_KEY` (live, marked Secret)
- **AC-5**: Žiadny secret v repo histórii (audit cez `git log -p` grep `sk_live` / `whsec_`).
- **AC-6**: Audit script `scripts/check-bundle-no-trackers.sh` rozšírený o whitelist Stripe domén a check že `STRIPE_SECRET_KEY` / `SUPABASE_SERVICE_ROLE_KEY` nikdy nie sú v `dist/client/`.
- **AC-7**: Story doc obsahuje step-by-step návod ako vygenerovať Stripe webhook secret (cez Stripe CLI alebo dashboard).

# E11.2 — `/podakovanie/$token` thank-you + invoice download

**Epic:** [E11 — Sponsorship UI + invoicing](../PLAN-2026-04-26-custom-tests-sponsorship.md#epic-11--sponsorship-ui--invoicing)
**Effort:** `S` (≤ ½ dňa)
**Priority:** `P2` (should)
**Status:** ✅ Done — token = Stripe Checkout Session ID (no extra column), `donation-status.ts` joins via `payment_intent_id`, polling 3s × 30s for webhook eventual delivery, `customer-portal.ts` issues Stripe-hosted Billing Portal session for monthly cancel.

## Kontext

Po úspešnej platbe — krátka thank-you page so zhrnutím, link na PDF
faktúry (Stripe-generated), info o tom, že subscription možno zrušiť
kedykoľvek cez Customer Portal.

## User story

> Ako sponzor, ktorý práve daroval, **chcem** vidieť potvrdenie a stiahnuť
> si faktúru, **aby** som mal písomný doklad pre účtovníctvo.

## Acceptance criteria

- **AC-1**: Route `/podakovanie/$token` (token = stripe checkout session ID alebo dedikovaný UUID v `donations`).
- **AC-2**: Loader → `supabase.from('donations').select(...)` cez stripe_payment_intent_id alebo session id.
  - Ak sa nenájde → „Hľadáme tvoju platbu… (Stripe webhook môže trvať pár sekúnd)" s auto-refresh každé 3s, max 30s, potom contact link.
- **AC-3**: Po nájdení:
  - „Ďakujeme {donor_label or 'za podporu'}!" hero
  - Sumár: čiastka, jednorazovo/mesačne, dátum
  - Tlačidlo „Stiahnuť faktúru (PDF)" → `donation.invoice_pdf_url` (Stripe-hosted)
  - Pre subscription: „Spravovať odbery" → Stripe Customer Portal link (server-generated)
- **AC-4**: Krátky text o transparentnosti: „Tvoje peniaze idú na hosting (Cloudflare), Supabase a tvorbu nového obsahu."
- **AC-5**: SEO: `<title>Ďakujeme za podporu — subenai</title>`, `noindex` (každý token unikátny).
- **AC-6**: Bezpečnosť: žiadne donor PII v URL, len opaque token.

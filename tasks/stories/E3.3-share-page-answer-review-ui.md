# ~~E3.3 — `/r/$shareId` — Answer review UI~~ ✅

**Epic:** [E3 — Post-test answer review](../PLAN-2026-04-25-rast-a-vzdelavanie.md#epic-3)
**Effort:** `M` (≤ 2 dni)
**Priority:** `P1` (must)
**Status:** ✅ Done (2026-04-26)

## Kontext

Share page `/r/$shareId` aktuálne ukazuje len skóre/personality/
breakdown. Po E3.1 budú odpovede v DB, po E3.2 máme zdieľaný
feedback komponent. Teraz pridáme UI pre detail review.

## User story

> Ako majiteľ share linku, **chcem** kliknúť tlačidlo
> "Pozri si moje odpovede" a prejsť všetkých 10 otázok s mojím
> výberom + správnou odpoveďou + vysvetlením, **aby** som sa
> naučil/a kde som chyboval/a.

## Acceptance criteria

- **AC-1**: `/r/$shareId` route fetchne `attempt.answers` z DB
  (Zod parse cez `AnswerRecordPersistedSchema`).
- **AC-2**: Pod existujúcim výsledkom sa zobrazí collapsible sekcia
  **"Pozri si moje odpovede (10)"** — defaultne **zatvorená**,
  expand iba na klik.
- **AC-3**: Po expand sa lazy-renderne 10 kariet, každá obsahuje:
  - Question prompt (z `getQuestionById(answer.questionId)`)
  - Visual block (cez existujúci `<VisualBlock>`)
  - Highlight na option ktorú používateľ vybral (border-primary)
  - Highlight na option ktorá bola správna (border-success +
    ✅, ak je iná než vybraná)
  - `<AnswerFeedback question={q} selectedId={a.optionId}
    mode="review" />` blok pod options
  - Response time chip ("Odpovedaj za 1.4s") — poznámka, ak
    response > 90% time limit, zafarbiť varovne.
- **AC-4**: Ak `attempt.answers.length === 0` (legacy záznam) →
  zobrazí sa friendly placeholder card "Detail odpovedí nie je
  dostupný pre staré výsledky" — žiadny crash.
- **AC-5**: SSR-friendly: pri prvom server-render sa zobrazí
  iba placeholder skeleton (otázky sú v JSON v DB, ale ich plný
  obsah z `questions.ts` sa loaduje cez bundle — akceptujeme
  klient-side hydration). Žiadne client-only flash error.
- **AC-6**: Lazy load: collapsed stav nerendruje 10 kariet do DOM
  (perf — bundle obsahuje VisualBlock pre 6 typov, šetríme work).
- **AC-7**: Smooth scroll na top kariet po expand (UX).
- **AC-8**: A11y: `<details>` element alebo `aria-expanded` na
  toggle button; screen reader announce "10 odpovedí dostupných".
- **AC-9**: Žiadny vizuálny shift score-summary po expand (CLS = 0).

## Subtasks

### Implementation
- [ ] V `src/lib/quiz/questions.ts` pridať helper
  `getQuestionById(id: string): Question | null` (ak chýba).
- [ ] V `src/routes/r.$shareId.tsx`:
  - Po fetchu `attempt`, parse `attempt.answers` cez Zod.
  - Render collapsible sekciu (button + state).
  - Render 10 `<AnswerReviewCard>` komponentov.
- [ ] Vytvoriť `src/components/quiz/AnswerReviewCard.tsx`:
  - Props: `answer: AnswerRecord`, `question: Question`
  - Render: prompt → visual → options s highlightom →
    `<AnswerFeedback mode="review" />` → time chip
- [ ] Defenzívne: ak `getQuestionById(answer.questionId)`
  vráti `null` (otázka odstránená z banky neskôr) → render
  fallback "Otázka už nie je dostupná" pre tú jednu kartu.

### Tests
- [ ] **Render test** (`tests/routes/r-shareId.test.tsx`):
  - mock attempt s 10 answers → render → assert collapsible
    button viditeľný, 10 kariet **NIE** v DOM (lazy)
  - klik button → assert 10 kariet v DOM
  - assert každá karta má `<AnswerFeedback>` v `mode="review"`
- [ ] **Backward compat test**: mock attempt s `answers=[]` →
  expand → assert placeholder card, žiadny render error.
- [ ] **Edge case test**: mock attempt kde 1 z 10 questionId
  neexistuje v `questions.ts` → assert ostatných 9 kariet
  v poriadku, 1 fallback.
- [ ] **A11y test**: `aria-expanded` toggluje true/false podľa
  state, button má `aria-controls` na sekciu.

### Documentation

- [ ] Pridať odkaz "Pozri si svoje odpovede" do `ResultsView`

### Code review (fresh context)

## Závislosti

- **Blokuje:** E3.1 (DB stĺpec), E3.2 (zdieľaný komponent)

## Riziká

| Riziko | Mitigácia |
|---|---|
| Bundle bloat z eager-loadu visual komponentov | Lazy import `<AnswerReviewCard>` cez `React.lazy` + `<Suspense>` |
| Otázka odstránená z banky → dangling reference | Per-card try/catch fallback (AC-4 + edge case test) |
| Smooth scroll konfliktuje s anchor links | Použiť `scrollIntoView({ behavior: "smooth", block: "start" })` na expand container |

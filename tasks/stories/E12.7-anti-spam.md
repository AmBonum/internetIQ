# E12.7 — Anti-spam pre respondent intake

**Epic:** [E12 — Education mode](../PLAN-2026-04-26-custom-tests-sponsorship.md#epic-12--education-mode-autori-zbierajú-výsledky)
**Effort:** `S` (≤ ½ dňa)
**Priority:** `P2` (should)
**Status:** ✅ Done (2026-05-01)

## Kontext

Edu mode má verejný link kde každý môže odoslať name+email. Bez
ochrany útočník vie automatizovane vyplniť falošné mená do tisícov
záznamov a poškodiť kvalitu dát autorovi.

## User story

> Ako autor edu testu, **chcem** spoliehať na to, že záznamy v dashboarde
> sú reálni respondenti, **aby** moje vyhodnotenie nebolo skreslené botmi.

## Acceptance criteria

- **AC-1**: **Honeypot field** v intake forme (E12.3 AC-2):
  - HTML: `<input name="hp_url" aria-hidden="true" tabindex="-1"
    autocomplete="off" style="position:absolute;left:-9999px">`
  - Server: ak `hp_url` nie je prázdny → reject 400 (silent, log only)
- **AC-2**: **Rate limit** na `/api/begin-edu-attempt`:
  - 3 attempts / 5 minút / IP / set_id (CF Pages native rate limiter)
  - 50 attempts / hodina / set_id (per-test cap, bráni masívnemu spamu)
  - Pri prekročení 429 s message „Príliš veľa pokusov, skús o pár minút"
- **AC-3**: **Email format validation**:
  - Server-side regex `^[^@\s]+@[^@\s]+\.[^@\s]+$` (presne ako CHECK
    constraint v E12.1)
  - **Bez MX lookup** — overhead pre MVP; bounce-y manažuje autor sám
  - Žiadny throw-away email blacklist (10minutemail, atď.) v MVP —
    autor vidí ktoré emaily sú zjavne fake a zmaže
- **AC-4**: **Duplicitná detekcia per (set_id, email)**:
  - Pri INSERT attempts kde existuje row s rovnakým `test_set_id +
    respondent_email`, vráť 409 Conflict s message „Už si test absolvoval"
  - Autor si môže pridať „povoliť opakované pokusy" toggle v composer-i
    (E12.2 extension — defer to v2 ak treba)
- **AC-5**: **Logovanie pokusov**:
  - Každý reject sa loguje do CF Pages logs (IP, user-agent, set_id, reason)
  - Žiadne PII v logoch (email iba prvé 3 znaky + `***@domain`)
- **AC-6**: **Captcha** — len ako fallback ak rate limit niekde zlyhá.
  Cloudflare Turnstile (free, žiadny tracking). Aktivuje sa cez env flag,
  default OFF v MVP, povolené pri abuse incident.
- **AC-7**: Mobile-first + a11y compliance — honeypot field neporušuje
  screen reader flow (skip cez `aria-hidden`).

## Subtasks

### Implementation
- [x] Honeypot field v intake forme: `aria-hidden="true"`, `tabindex="-1"`, `position:absolute; left:-9999px`. Server: silent reject + warn-log s IP + set_id (žiadne PII).
- [x] Rate-limit v `/api/begin-edu-attempt`: per-IP 3/5min + per-set 50/h (oba musia prejsť, žiadne fall-through). Implementuje `ipRateLimit.consume()` z `_lib/security`.
- [x] Email regex `^[^@\s]+@[^@\s]+\.[^@\s]+$` — server-side; identický s `attempts_respondent_email_format` CHECK z E12.1.
- [x] Duplicitná detekcia per (set_id, email_lc) cez service-role SELECT — vráti 409 `already_attempted`.
- [x] PII redaction v logoch — `redactEmail()` helper (3 znaky local + `***@domain`).
- [x] Bonus: `/api/finish-edu-attempt` má aj svoj rate-limit (6/5min/IP) — chráni proti zlomenej finish-loop attacku.
- [ ] Turnstile integration — defer pre v2 (env-flag gated, aktivovať len pri abuse incidente).

## Riziká

| Riziko | Mitigácia |
|---|---|
| Rate limit zlyhá pri DDOS | Cloudflare DDoS protection na úrovni edge zachytí pred funkciou |
| Duplikátny check zachytí legit re-take | Autor môže manuálne vymazať starý záznam cez dashboard, alebo my pridáme `allow_retake` toggle vo v2 |
| Turnstile v EÚ vyžaduje cookie consent | Turnstile sám nepoužíva cookies — privacy-friendly fingerprintless challenge |

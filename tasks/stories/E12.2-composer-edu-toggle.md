# E12.2 — Composer education toggle + author password setup

**Epic:** [E12 — Education mode](../PLAN-2026-04-26-custom-tests-sponsorship.md#epic-12--education-mode-autori-zbierajú-výsledky)
**Effort:** `M` (≤ 2 dni)
**Priority:** `P2` (should)
**Status:** ✅ Done (2026-05-01)

## Kontext

Composer (E8.2) potrebuje rozšírenie pre education mode — toggle „Zbierať
odpovede" + heslo authora pred uložením test_setu. Bez toho autor nemá
ako sa neskôr autentifikovať pri prístupe k výsledkom.

## User story

> Ako učiteľ pripravujúci test, **chcem** v composer-i zapnúť možnosť
> zbierať odpovede a nastaviť si heslo, **aby** som po teste mohol
> stiahnuť výsledky bez registrácie u nás.

## Acceptance criteria

- **AC-1**: V composer-i (`/test/zostav`) v sekcii „Settings" pridaný
  toggle **„Zbierať odpovede s menom a emailom"** (default OFF). Tooltip
  vysvetľuje, že odpovede sú PII a viažu sa na samostatný consent.
- **AC-2**: Keď toggle ON, zobrazí sa nový password input:
  - Label: „Heslo na pozeranie výsledkov"
  - Validácia: minimum 8 znakov
  - Tlačidlo „Zobraziť" prepína type=password ↔ type=text
  - Helper text: „Toto heslo budeš potrebovať na otvorenie výsledkov.
    Ak ho stratíš, výsledky nezískaš späť. Žiadne reset cez email."
- **AC-3**: Po submit composer:
  - Browser hashne heslo cez bcrypt (`bcryptjs` knižnica, salt rounds 10)
  - POST do `/api/test-sets` s `collects_responses: true, author_password_hash: '$2a$10$...'`
  - Server validuje že hash má bcrypt formát; ukladá ako-je
- **AC-4**: Po vytvorení dostane autor 2 linky:
  - **Public link** (na zdieľanie respondentom): `/test/zostava/{id}`
  - **Results link** (s password gate): `/test/zostava/{id}/vysledky`
  - Confirm dialog: „Skopíruj si oba linky aj heslo PRED tým, ako zatvoríš túto stránku."
- **AC-5**: Default toggle OFF — žiadny user nedostane PII collection
  „omylom"; education mode je vedomý opt-in.
- **AC-6**: A11y — toggle a password input majú ARIA labels, error states
  čítateľné screen readerom.

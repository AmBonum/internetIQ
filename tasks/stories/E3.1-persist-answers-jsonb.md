**Epic:** [E3 — Post-test answer review](../PLAN-2026-04-25-rast-a-vzdelavanie.md#epic-3)

**Priority:** `P1` (must)
**Status:** ✅ Done (2026-04-26)

## Kontext

Aktuálne `attempts` tabuľka neobsahuje individuálne odpovede —
len agregované `breakdown`, `flags`, `insights`, `stats`. Bez
persistovaných odpovedí nie je možné po reload-e share linku
zobraziť detail "ktorú možnosť som vybral pri otázke 3".
Bez DB persistencie by review fungoval iba v session pamäti
(zaniká pri F5), čo je horšie UX a stratíme cennú analytiku
"ktoré otázky najčastejšie chytajú ľudí".

## User story

> Ako návštevník otvárajúci share link svojho výsledku, **chcem**
> vidieť moje konkrétne odpovede aj po refreshi alebo na inom
> zariadení, **aby** som sa mohol/a poučiť z chýb.

## Acceptance criteria

- **AC-1**: Migrácia v `supabase/migrations/{ts}_attempt_answers.sql`
  pridá `answers JSONB NOT NULL DEFAULT '[]'` do `attempts`.
- **AC-2**: Zod schema v `src/lib/quiz/schema.ts` popisuje
  `AnswerRecordPersistedSchema`:
  ```ts
  z.object({
    questionId: z.string(),
    optionId: z.string().nullable(), // null = timeout
    correct: z.boolean(),
    severity: z.enum(["critical", "medium", "minor"]).nullable(),
    responseMs: z.number().int().min(0),
    category: z.enum(["phishing", "url", "fake_vs_real", "scenario"]),
    difficulty: z.enum(["easy", "medium", "hard"]),
  })
  ```
- **AC-3**: `ResultsView.persistResult()` posiela `answers` array
  do INSERT-u. Vyžaduje rozšírenie `ScoreResult` o `answers:
  AnswerRecord[]` (alebo prijatie `answers` ako separátneho parametra
  do `persistResult` — preferované, menej coupling).
- **AC-4**: Existujúce riadky bez `answers` (pre-migrácia, default
  `[]`) zobrazia v review fallback "Detail odpovedí nie je dostupný
  pre staré výsledky" — žiadny crash.
- **AC-5**: Auto-generované TS typy reflektujú `answers` field
  ako `Json` (Supabase generic typ); aplikačná vrstva ich parsuje
  cez Zod schema do silne typovaného array.
- **AC-6**: GIN index na `answers` (`CREATE INDEX
  idx_attempts_answers ON attempts USING gin (answers)`) — pripraví
  pôdu pre budúce queries typu "ktoré optionId najčastejšie zlyháva".

## Riziká

| Riziko | Mitigácia |
|---|---|
| PII leak — `optionId` obsahuje free-text | Question options sú konečná enumerácia (`a`, `b`, `c`, `d`), free-text neexistuje |
| JSONB schema drift po refaktore questions.ts | Zod schema je single source of truth, runtime parse pri read |
| Storage rastie nelineárne pri škálovaní | 10 answers × ~150 B = 1.5 KB / attempt; pri 100k attempts = 150 MB → bohato v free tier |
| GIN index drahý pri INSERT | INSERTs sú rare (1 per test completion); read-heavy workload odôvodní index |

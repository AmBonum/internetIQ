# E7.1 — Test pack content schema + registry

**Epic:** [E7 — Industry test packs](../PLAN-2026-04-26-custom-tests-sponsorship.md#epic-7--industry-test-packs)
**Effort:** `S` (≤ ½ dňa)
**Priority:** `P1` (must)
**Status:** 🟡 Ready

## Kontext

Industry packy sú statické TS moduly (rovnaký pattern ako kurzy). Aby
sme mali typovo bezpečnú štruktúru, definujeme schému najprv. Schéma
je single source of truth — každá zmena je breaking pre všetkých
autorov packov.

## User story

> Ako vývojár pridávajúci nový industry pack, **chcem** typovo bezpečnú
> štruktúru, **aby** sa nezabudlo na povinné polia (slug, title,
> question IDs, passing threshold).

## Acceptance criteria

- **AC-1**: `src/content/test-packs/_schema.ts` exportuje:
  ```ts
  export type Industry =
    | "eshop" | "gastro" | "autoservis" | "it" | "verejne_sluzby"
    | "dispecing" | "doprava" | "marketing" | "zdravotnictvo" | "skoly"
    | "strojova_vyroba" | "pneuservis" | "sme_ucto" | "horeca" | "servis";

  export type TestPack = {
    slug: string;            // [a-z0-9-]+
    title: string;
    tagline: string;
    industry: Industry;
    industryEmoji: string;
    targetPersona: string;   // 1 veta — komu je pack určený
    questionIds: string[];   // ručne kurátorovaný zoznam, 8–25 položiek
    honeypotRatio?: number;  // 0..1 — odporúčaný podiel honeypot otázok pri náhodnom doplnení
    passingThreshold: number; // 0..100, default 70
    publishedAt: string;     // ISO 8601
    updatedAt: string;
    sources?: { label: string; url: string }[];
  };
  ```
- **AC-2**: `src/content/test-packs/index.ts` exportuje:
  ```ts
  export const TEST_PACKS: TestPack[] = [/* statické imports */];
  export function getPackBySlug(slug: string): TestPack | null;
  export function getPacksByIndustry(industry: Industry): TestPack[];
  export function listPublishedPacks(): TestPack[];
  ```
- **AC-3**: Slug uniqueness assertion v build-time (rovnako ako pri kurzoch).
- **AC-4**: Zod schéma `testPackSchema` v rovnakom súbore.
- **AC-5**: Pack autoring guide `tasks/E7-pack-authoring-guide.md` — krátky manuál.
- **AC-6**: Schéma má cross-validáciu cez Zod refine: `questionIds.length >= 8 && questionIds.length <= 25`.
- **AC-7**: Helper `validatePackQuestionIds(pack)` overí že každý ID v `questionIds` existuje v `QUESTIONS` poolu (bezpečnostná sieť pri merge).

## Subtasks

### Implementation
- [ ] Vytvoriť `src/content/test-packs/_schema.ts` s TS types + Zod schema.
- [ ] Vytvoriť `src/content/test-packs/index.ts` s registry + helpers.
- [ ] Vytvoriť `src/content/test-packs/_template.ts` ako referenčnú šablónu (nie registrovaná).

### Tests
- [ ] **Schema test** (`tests/content/test-packs-schema.test.ts`):
  - každý pack v `TEST_PACKS` parsuje cez `testPackSchema`
  - slug uniqueness
  - každý `questionIds[*]` existuje v `QUESTIONS` poolu
  - `8 <= questionIds.length <= 25`
  - `passingThreshold` v range 0..100
- [ ] **Helper test**:
  - `getPackBySlug` happy + null path
  - `getPacksByIndustry` filter správne

### Documentation
- [ ] `tasks/E7-pack-authoring-guide.md` — workflow + príklad packu.
- [ ] JSDoc na hlavných typoch.

### Code review (fresh context)
- [ ] Reviewer overí ergonómiu schémy (≤ 30 riadkov pre nový pack).
- [ ] Reviewer overí že schéma je rozšíriteľná (nový industry = jedna union položka).

## Závislosti

- E9.1, E9.2, E9.3, E9.4 — banky musia obsahovať dosť otázok pre packy
  (technicky nie hard-blocker pre schému, ale bez nich nemajú packy z čoho čerpať).

## Riziká

| Riziko | Mitigácia |
|---|---|
| Pack tagne neexistujúci `questionId` po refaktore questions.ts | `validatePackQuestionIds` test fail-uje na build-e |
| Industry enum prerastie na 30+ položiek | Limit na 15–20 industry buckets, väčšie pokrytie cez sub-tagging v budúcnosti |

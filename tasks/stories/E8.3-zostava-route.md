# E8.3 — `/test/zostava/$id` shareable saved set

**Epic:** [E8 — Manual test builder](../PLAN-2026-04-26-custom-tests-sponsorship.md#epic-8--manual-test-builder)
**Effort:** `S` (≤ ½ dňa)
**Priority:** `P2` (should)
**Status:** ⛔ Blocked by E8.1, E8.2

## Kontext

Po stlačení „Zdieľať zostavu" v builderi sa test sety uloží do DB s
UUID. Tento route ho načíta a pustí ako pack.

## User story

> Ako prijímateľ zostava-linky, **chcem** otvoriť ju a hneď spustiť
> test, **aby** som videl, či splním sadu, ktorú mi niekto poslal.

## Acceptance criteria

- **AC-1**: Route `src/routes/test.zostava.$id.tsx`:
  - loader → `supabase.from('test_sets').select().eq('id', params.id).single()`
  - 404 ak nie je
- **AC-2**: Intro screen:
  - Hero: `creator_label` ak existuje (napr. „E-shop Q1 2026 onboarding"), inak generický „Pripravený test pre teba"
  - Sub-text: počet otázok + „Vyhovenie pri skóre ≥ {passing_threshold}"
  - Source pack mention ak `source_pack_slugs` nie je prázdny: „Zostavené z packov: E-shop, Marketing"
  - Start button
- **AC-3**: Po štarte → `TestFlow` v override móde s `questions_override` z `question_ids` a `passing_threshold` v override.
- **AC-4**: ResultsView ukáže „Vyhovuje pre tento test" badge ak `finalScore >= testSet.passing_threshold`. Žiadny industry-specific text — len generický „Vyhovuje pre tento test".
- **AC-5**: SEO: `<title>{creator_label || 'Zdieľaný test'} — subenai</title>`, robots noindex (každý ID je unique, indexovať netreba).
- **AC-6**: Po dokončení testu — share link komunikuje výsledok ako bežný `attempts` riadok (s `share_id` v Supabase). Attempts row pridáva voliteľný odkaz `test_set_id` (nullable foreign key) pre analytickú väzbu.

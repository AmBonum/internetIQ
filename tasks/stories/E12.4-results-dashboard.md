# E12.4 — `/test/zostava/$id/vysledky` results dashboard

**Epic:** [E12 — Education mode](../PLAN-2026-04-26-custom-tests-sponsorship.md#epic-12--education-mode-autori-zbierajú-výsledky)
**Effort:** `M` (≤ 2 dni)
**Priority:** `P2` (should)
**Status:** ✅ Done (2026-05-01)

## Kontext

Hlavná value proposition pre školy/firmy — autor po teste otvorí
dashboard, zadá heslo a vidí ako jeho trieda / tím absolvovala. Bez
neho je celá education feature bezúčelná.

## User story

> Ako autor edu testu, **chcem** cez password-protected stránku vidieť
> kto absolvoval test, koľko bodov získal a kde sú slabiny, **aby** som
> ohodnotil študentov a vedel kam zamerať ďalšiu výuku.

## Acceptance criteria

- **AC-1**: Route `/test/zostava/$id/vysledky` (file `src/routes/test.zostava.$id.vysledky.tsx`).
- **AC-2**: Pri prvom load: password gate form (1 input + submit). Žiadny
  iný obsah viditeľný pred autentifikáciou.
- **AC-3**: Submit → POST `/api/verify-author-password` (CF Pages Function):
  - Body: `{ set_id, password }`
  - Server: rate limit 5/15min/IP/set_id (E12.7 pattern), volá Supabase
    RPC `verify_test_set_password(set_id, password)`, na success vystaví
    JWT (`results_token`, 60 min TTL) v cookie httpOnly+secure+sameSite=lax
  - Reject: 401, generic error message (anti-enumeration)
- **AC-4**: Po success cookie set, redirect na rovnakú URL → loader prečíta
  cookie, validuje JWT cez `/api/results-data` POST a získa results JSON.
- **AC-5**: Dashboard layout:
  - **Header**: pack title + creator_label + počet respondentov
  - **Aggregate stats**:
    - Priemerné skóre + min + max + medián (sparkline trend cez čas)
    - Distribúcia (histogram 0-25, 25-50, 50-75, 75-100)
    - Pass rate (% nad passing_threshold)
    - Per-question accuracy (top 3 najťažšie + top 3 najľahšie otázky)
  - **Per-respondent table**:
    - Stĺpce: Meno, Email, Skóre, Percentil, Dátum, Vyhovel? (badge), Akcie
    - Sortable cez headers
    - Akcie: „Vymaž respondenta" (potvrdenie + DELETE attempts row + log audit)
    - Search input (filter cez meno/email)
  - **Export**: CSV download tlačidlo (klient-side z dát, žiadny server roundtrip)
- **AC-6**: Empty state — „Zatiaľ žiadne odpovede. Zdieľaj link {public_url}."
- **AC-7**: Logout button — vymaže cookie, redirect na intro.
- **AC-8**: A11y — table má `<caption>`, `scope` atribúty na th, sortable
  headers majú `aria-sort`.
- **AC-9**: Bezpečnosť:
  - JWT secret v `JWT_SECRET` env (zhodne s E12.3)
  - Cookie httpOnly + secure + sameSite=lax + Path=/test/zostava/$id/vysledky
  - CSRF nie je problém — žiadny state-changing request po login okrem
    `verify-author-password` (origin-checked)
- **AC-10**: Mobile-first. Tabuľka sa scroll-snapuje horizontálne.

## Riziká

| Riziko | Mitigácia |
|---|---|
| Brute-force heslá | rate limit + bcrypt cost faktor 10 (slow comparison) |
| Token leak cez log | session token nikdy do logov; JSON.stringify mimo error paths |
| Veľký test_set s 1000+ respondentmi pomalý | server-side paging (50/page) v results-data API, klient lazy-load |

# AH-11.3 — Privileged createServerFn handlers with supabaseAdmin

**Epic:** [AH-11 — Mock → Supabase wiring + production-ready verification](../PLAN-2026-05-17-admin-hub-integration.md)
**Effort:** `L` (≤ 1 week)
**Priority:** `P1` (must)
**Status:** Backlog

## Goal

Implement the privileged server-only operations that AH-7 left as mock
TODOs: user/role management, audit-log inserts, anonymization, DSR
processing. All run through `createServerFn` handlers in `*.server.ts`
files and use `supabaseAdmin` (service role).

## Acceptance criteria

- All ported files have `npx eslint --fix` applied; `npm run lint` is 0/0.
- `supabaseAdmin` imported ONLY in `*.server.ts` files (enforced by the
  ESLint `no-restricted-imports` rule added in AH-1.7); `git grep
  "supabaseAdmin" src/` lists only `.server.ts` files.
- `git grep VITE_SUPABASE_SERVICE src/` returns zero matches.
- Every privileged operation inserts a row into `audit_log` with the
  correct `action`, `actor_user_id`, and `pii_access` flag (true for any
  read of identifiable respondent data, role changes, or DSR fulfilment).
- Server fns run the auth middleware (AH-3.1) and role middleware
  (AH-10.1) in sequence; unauthenticated → 401, non-admin → 403.
- No `data-testid` or i18n changes (server-only story).
- CONSENT_VERSION unchanged (1.4.0 from AH-7.1).

## Implementation

- New server fns under `src/lib/admin/<area>.functions.ts`, one file per
  privileged group (`users.functions.ts`, `roles.functions.ts`,
  `audit.functions.ts`, `anonymize.functions.ts`, `dsr.functions.ts`).
- Reuse `src/integrations/supabase/auth-middleware.ts` and
  `src/integrations/supabase/role-middleware.ts` (exist from AH-3.1 / AH-10.1).
- Anonymization fn matches the `purge_expired_respondent_pii` semantics
  but is invoked explicitly (cron stub is separate, AH-1.8).
- DSR fulfilment writes status transitions and inserts an audit row per
  action; rejection requires a reason.

## Tests

- `tests/lib/admin/<area>.functions.test.ts` — Vitest unit tests per
  server fn with `supabaseAdmin` mocked; cover the happy path, the 401
  path, the 403 path, and the audit-row assertion.
- `e2e/integration/admin-hub/privileged-supabase.spec.ts` gated on
  `SUPABASE_INTEGRATION=1`, exercising one end-to-end role-grant + audit
  trail.

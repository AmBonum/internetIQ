## Kontext

Aktuálne share UI ponúka iba (a) Web Share API tlačidlo,
(b) Copy link, (c) IG Story PNG download. Web Share API
nieje k dispozícii vo väčšine desktop browserov → user na
desktope má iba "Copy link", čo je vyššia friction (ručne
prelepiť, otvoriť FB/X, vložiť, write copy). Native share
intent linky platforiem dovoľujú **jeden klik** na desktope
aj mobile, bez 3rd-party SDK / pixelov / consent zmeny.

UTM parametre na share linkoch nám neskôr (po analytics
implementácii) povedia ktorá sieť privádza najviac návštev —
input pre rozhodnutie kam investovať promo.

## User story

> Ako používateľ na desktope alebo mobile, **chcem** klikom
> jedného tlačidla zdieľať svoj výsledok na Facebook /
> Messenger / WhatsApp / X / LinkedIn / Telegram s preformátovaným
> textom, **aby** som nemusel/a kopírovať a paste-ovať.

## Acceptance criteria

- **AC-1**: V `ResultsView` pod existujúce tlačidlá
  "📲 IG Story PNG" + "📤 Zdieľaj link" pridať grid 6
  share-platform buttonov:
  - **Facebook** — `https://www.facebook.com/sharer/sharer.php?u={url}`
  - **Messenger** — `https://www.facebook.com/dialog/send?app_id={id}&link={url}&redirect_uri={url}`
    (desktop) / `fb-messenger://share?link={url}` (mobile UA detect)
  - **WhatsApp** — `https://api.whatsapp.com/send?text={text}%20{url}`
  - **X** — `https://twitter.com/intent/tweet?text={text}&url={url}`
  - **LinkedIn** — `https://www.linkedin.com/sharing/share-offsite/?url={url}`
  - **Telegram** — `https://t.me/share/url?url={url}&text={text}`
- **AC-2**: Každý button otvára nové okno cez
  `window.open(intentUrl, "_blank", "noopener,noreferrer,width=600,height=600")`.
- **AC-3**: Share **text** (slovak):
  `"Mám Internet IQ {score}/100. Lepší než {pct}% ľudí. Som {personality}. Skús aj ty:"`
  — encoded cez `encodeURIComponent`.
- **AC-4**: **UTM parametre** sa pripoja na `shareUrl`:
  `${shareUrl}?utm_source={platform}&utm_medium=share&utm_campaign=results`
  — kde `platform ∈ { facebook, messenger, whatsapp, x, linkedin, telegram }`.
  Helper musí korektne handleovať prípad keď `shareUrl` už obsahuje `?`
  (použiť `&` namiesto `?`).
- **AC-5**: Vizuál — 3×2 grid na desktope, 2×3 na mobile.
  Každý button = SVG icon platformy (inline, nie external set)
  + label. Rovnaký button-style framework ako existujúce
  share buttons (rounded-xl, border-2 border-border, hover:border-primary/60).
- **AC-6**: **Žiadne 3rd-party SDK / pixely** sa neloadujú —
  čisté `window.open` na intent URL. Žiadny FB Pixel, žiadny
  X widget script. Audit cez `npm run build` + grep "facebook.net"
  v output musí vrátiť 0 hits.
- **AC-7**: **Web Share API tlačidlo sa zachová** — `navigator.share`
  je často preferovaná cesta na mobile (one-tap, native dialog).
  Share grid je secondary fallback / power-user option.
- **AC-8**: Helper modul `src/lib/share/intents.ts` exportuje:
  ```ts
  export type SharePlatform =
    | "facebook" | "messenger" | "whatsapp"
    | "x" | "linkedin" | "telegram";

  export const SHARE_PLATFORMS: ReadonlyArray<{
    id: SharePlatform; label: string; icon: ReactNode;
  }>;

  export function buildShareIntentUrl(
    platform: SharePlatform,
    shareUrl: string,
    text: string,
  ): string;

  export function withUtm(
    baseUrl: string,
    source: SharePlatform,
    medium?: string,
    campaign?: string,
  ): string;
  ```
- **AC-9**: **UTM consent gate** — keď používateľ pristúpi
  cez share link s UTM parametrami, helper `readUtmFromUrl()`
  v `src/lib/tracking.ts` prečíta params **iba ak**
  `isAllowed("analytics") === true`. Bez consent → no-op,
  žiadny analytics event sa neodošle. Toto AC popisuje
  read-side správanie; samotná telemetria sa aktivuje
  až keď bude analytics endpoint pripojený (out-of-scope).
- **AC-10**: Komponent `<SocialShareGrid>` je samostatný file
  `src/components/quiz/SocialShareGrid.tsx`, používa sa aj
  v `ResultsView` aj v `r.$shareId.tsx` (príjemca môže
  re-shareovať).
- **AC-11**: Messenger button na desktope detekuje cez UA
  ("`Mobi`" / mobile navigator) — ak mobile, použije
  `fb-messenger://` deep link s fallbackom na FB dialog ak
  scheme nieje handled (`window.open` zlyhá → po 1.5s
  redirect na FB dialog). Ak desktop → priamo FB dialog.
  *Voliteľné: ak je to príliš krehké, použiť FB dialog na oboch.*

## Riziká

| Riziko | Mitigácia |
|---|---|
| Platforma zmení intent URL formát | Helper je centralizovaný (1 file), update jeden — pokrytý unit testami |
| Pop-up blocker na desktope znemožní `window.open` | Akceptovateľný UX — user dovolí pop-up. Alternatíva (target="_blank" anchor) má rovnaký efekt |
| Messenger deep link `fb-messenger://` zlyhá ak appka nieje | Fallback na FB dialog po 1.5s timeout; AC-11 môžme zjednodušiť na desktop-only FB dialog ak je to príliš krehké |
| FB Sharer otvorí "log in" page namiesto share dialógu pre nelogovaných | FB-controlled, mimo nášho dosahu. Existing `sharer.php` URL ostáva industry standard |
| UTM v URL roztrhne SEO equity (canonical) | `r.$shareId.tsx` route by mala mať `<link rel="canonical">` bez UTM (E5.12 SEO story to už pokrýva — koordinovať) |
| User používa adblock ktorý blokne `sharer.php` | Outside our control; share grid je addon, primárny share flow (Web Share API + Copy link) ostáva funkčný |

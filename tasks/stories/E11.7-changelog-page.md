# E11.7 — `/zmeny` deploy history / changelog

**Epic:** [E11 — Sponsorship UI + invoicing](../PLAN-2026-04-26-custom-tests-sponsorship.md#epic-11--sponsorship-ui--invoicing)
**Effort:** `S` (≤ ½ dňa)
**Priority:** `P2` (should)
**Status:** 🟡 Ready

## Kontext

Verejný changelog je trust signál: ukazuje že projekt je živý, aktívne
udržiavaný, a transparentne dokumentuje zmeny. Generuje sa
auto-build-time z `CHANGELOG.md` (Markdown) alebo z git tagov.

## User story

> Ako návštevník / sponzor, **chcem** vidieť čo sa v projekte za posledné
> obdobie zmenilo a kedy bol posledný deploy, **aby** som mohol
> objektívne posúdiť aktivitu projektu.

## Acceptance criteria

- **AC-1**: Vytvoriť `CHANGELOG.md` v root repa, formát Keep a Changelog 1.1
  ([keepachangelog.com](https://keepachangelog.com/en/1.1.0/)):
  ```markdown
  # Changelog

  All notable changes to this project will be documented in this file.

  The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
  and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

  ## [Unreleased]

  ## [1.5.0] — 2026-04-27
  ### Added
  - /podpora donate page (E11.1)
  - /o-projekte presentation page (E11.6)
  ### Changed
  - CONSENT_VERSION bumped to 1.2.0 (E10.5)
  ```
- **AC-2**: Build-time skript `scripts/parse-changelog.mjs` parsuje CHANGELOG.md
  do JSON štruktúry a uloží do `src/content/changelog.generated.json`:
  ```json
  [
    {
      "version": "1.5.0",
      "date": "2026-04-27",
      "added": ["/podpora donate page (E11.1)", ...],
      "changed": [...],
      "fixed": [...],
      "removed": [...]
    }
  ]
  ```
- **AC-3**: Skript `parse-changelog.mjs` zaradiť do `npm run build` pipeline (rovnako ako sitemap generator).
- **AC-4**: Route `/zmeny` (file `src/routes/zmeny.tsx`):
  - Loader: import generated JSON
  - Render zoznam verzií, najnovšia prvá
  - Per verzia: badge (`Pridané` zelený, `Zmenené` modrý, `Opravené` oranžový, `Odstránené` červený) + bullet list
  - Permalink na version: `/zmeny#v1.5.0`
- **AC-5**: SEO:
  - `<title>Zmeny a verzie — Internet IQ Test</title>`
  - canonical, robots index follow
  - Schema.org `Article` JSON-LD pre každú verziu (changelog as articles)
- **AC-6**: Footer link „Zmeny" pridaný.
- **AC-7**: Pri každom deploy (commit s `chore(release):` prefix) sa očakáva
  že developer aktualizuje `CHANGELOG.md`. Pre-commit hook upozorní ak
  nebol updated (mimo MVP, odporúčanie).
- **AC-8**: Slovak strings: „Pridané", „Zmenené", „Opravené", „Odstránené".
- **AC-9**: A11y: `<time datetime>` pre dátumy, semantic headings hierarchy.
- **AC-10**: Mobile-first. Verzie sa zobrazujú v reverse chronological order.

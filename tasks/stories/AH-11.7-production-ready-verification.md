# AH-11.7 — Production-Ready verification suite

**Epic:** [AH-11 — Mock → Supabase wiring + production-ready verification](../PLAN-2026-05-17-admin-hub-integration.md)
**Effort:** `M` (≤ 2 days)
**Priority:** `P0` (production-ready gate)
**Status:** Backlog

## Goal

Run every one of the 15 items from the master plan's
"Definition of Production-Ready" checklist and produce a single,
auditable verification report. AH-11 does not merge to `main` until
every item is green and the report is checked in.

## Acceptance criteria

- `tasks/AH-11-verification-report.md` exists and contains a numbered
  section for each of the 15 DoP-R items from
  `tasks/PLAN-2026-05-17-admin-hub-integration.md`.
- Every section has: command run, raw output (or short excerpt with
  link to CI artifact), explicit PASS/FAIL line, timestamp.
- All 15 items show PASS. Items below are non-negotiable PASS gates:
  1. `npm run lint` → 0/0
  2. `npm test` → all green, no `.skip` / `.only`
  3. `npm run build` → no errors or warnings
  4. `npm run e2e:browser` → all green (new `/app`, `/admin`, `/t/$shareId` included)
  5. `npm run e2e:integration` → all green
  6. `npm run audit:bundle` → PASS (from AH-11.6)
  7. Every new DB table has RLS enabled — output of cross-query in PLAN
  8. `git grep VITE_SUPABASE_SERVICE src/` → zero matches
  9. `/t/$shareId` incognito smoke completes without auth cookie
  10. `/admin` returns 403 for unauthenticated request
  11. `audit_log` PII-flagged entry written on `/admin/respondents` load
  12. DSR round-trip: `/app/legal/dsr` submit → `/admin/dsr` queue with `sla_due_at`
  13. Sitemap correct; `/app/*`, `/admin/*`, `/t/$shareId` absent; robots `Disallow` lines present
  14. Privacy page (`/privacy`) has retention rows for new PII categories
  15. CF Pages → Functions → Logs: zero unhandled 500s after 5-min smoke soak
- The story commit message links to the report file.

## Implementation

- Create `tasks/AH-11-verification-report.md` with the 15-section skeleton.
- For each section, run the command, paste relevant output (≤ 30 lines per
  section; link to longer artifacts).
- For browser-based checks (items 9, 10, 11, 12, 14), run via the existing
  Playwright suite plus targeted manual incognito smoke; capture screenshots
  into `tasks/AH-11-verification-screenshots/` and reference them.
- For items needing the CF Pages preview deploy (item 15), document the
  preview URL, the 5-minute soak window, and the empty error-log screenshot.
- For item 7, run the cross-query against the preview Supabase DB and paste
  the table list.
- For item 11, capture the `audit_log` row in the preview DB and paste the
  SELECT output.

## Tests

- The verification report itself is the test artifact. No new Vitest specs
  needed unless an item exposes a gap, in which case fix and re-verify.
- Optional: orchestration spec at `e2e/integration/production-ready.spec.ts`
  that wraps the green-path subset of items 4, 5, 9, 10, 12, 13 into a
  single CI job — gates on `process.env.PRODUCTION_READY_CHECK=1`.

## Documentation

- `tasks/AH-11-verification-report.md` (this story creates it).
- `tasks/FEATURE_MAP-admin-hub.md` — status columns for any rows still
  marked In Progress are bumped to Done.

## Code review

Fresh-context prompt: reviewer opens the report and confirms each of the
15 sections has a real command + real output + PASS. Reviewer also reruns
items 1–6 + 8 locally and confirms the output matches. Items 7, 9–15
require the preview env; reviewer either reruns them or trusts the
captured artifacts. Any FAIL blocks merge.

**Status:** ✅ Done — `/spravovat-podporu` form + `/api/portal-magic-link` (anti-enumeration: identical response regardless of customer existence). Footer "Podporiť projekt" link landed in E11.1.

## Kontext

Sekundárna ale dôležitá UX požiadavka — sponzorský footer link na
discoverability + canonical "spravovať odbery" CTA cez Stripe Customer Portal.
Žiadny custom self-service UI; Stripe portal je right tool.

## User story

> Ako mesačný sponzor, **chcem** kedykoľvek z hlavnej stránky nájsť možnosť
> spravovať / zrušiť odber, **aby** som sa nemusel hľadať v emailoch alebo
> kontaktovať support.

## Acceptance criteria

- **AC-1**: Footer (`src/components/Footer.tsx`) pridáva link „Podporiť projekt" → `/podpora`.
- **AC-2**: Privacy stránka pridáva sekciu „Ako spravovať / zrušiť odber" s linkom na Customer Portal flow:
  - User zadá email
  - Stripe pošle email s magic link
  - Otvorí self-service portal
- **AC-3**: Route `/spravovat-podporu` (pre user-friendly URL):
  - Form: „Zadaj email, ktorým si prispel/a"
  - Submit → POST na CF Pages Function `/api/portal-magic-link`
  - Server overí v Stripe Customer API ak existuje customer s tým emailom; **bez ohľadu na výsledok return 200** s rovnakou správou „Pošlite na váš email odkaz na správu odberu, ak ste vyznam s touto schránkou prispeli." → **anti-enumeration** (útočník nezistí, ktoré emaily sú sponzori).
  - Ak existuje → cez `billing_portal.sessions.create` vygeneruje URL → pošle cez E11.8 email infra (template `magic-link-portal.html`).
- **AC-4**: Žiadny dark pattern. „Zrušiť odber" je 1-click v Customer Portal-e.
- **AC-5**: A11y: form labels, error messages.
- **AC-6**: Rate limit per IP a per email (server-side): 3 requests / 10 minút. Po prekročení 429 s rovnakou anti-enumeration správou.
- **AC-7**: Magic link platnosť 1 hodina (Stripe default). Po uplynutí treba znova requestnut.

# ~~E4.3 — Integrácia `TrapDialog` do `ResultsView`~~ ✅

**Epic:** [E4 — Data-trap edukačný popup](../PLAN-2026-04-25-rast-a-vzdelavanie.md#epic-4)
**Effort:** `S` (≤ ½ dňa)
**Priority:** `P1` (must)
**Status:** ✅ Done (2026-04-26)

## Kontext

Komponent z E4.2 hotový a otestovaný. Teraz ho zapojíme do
`ResultsView` aby vyskočil v správnom momente a neminul prvú
emóciu (skóre + personality reveal). Načasovanie je dôležité —
príliš skoro a používateľ ešte nestihol prečítať skóre, príliš
neskoro a stratil pozornosť.

## User story

> Ako návštevník dokončujúci test, **chcem** dostať trap-popup
> hneď po výsledku (nie pred ním), **aby** som mohol najprv vidieť
> svoje skóre a až potom sa dať testovať na "ostrejšiu" obozretnosť.

## Acceptance criteria

- **AC-1**: `ResultsView` renderuje `<TrapDialog />` so stavom
  `open: boolean`. Default `open=false`.
- **AC-2**: Dialóg sa otvorí automaticky po **5 sekundách** od
  mountu `ResultsView` (timer cez `useEffect` + `setTimeout`).
  5s je heuristika — používateľ stihne prečítať skóre, ešte sa
  nepustil do scrollu k share buttonom.
- **AC-3**: Ak používateľ už raz dialóg videl
  (`localStorage.getItem("iiq_trap_seen") === "1"`), dialóg
  sa **neotvorí automaticky**, ale je dostupný cez tlačidlo
  "Vyskúšaj si pasce na osobné údaje" v ResultsView body.
- **AC-4**: Flag `iiq_trap_seen` sa nastaví **iba** keď používateľ
  klikne CTA "Som naivný, nech to už nikdy neurobím" — NIE pri
  pasívnom zatvorení (ESC, klik mimo). Tak druhý raz dostane
  ešte šancu.
- **AC-5**: Trap NEZASAHUJE share flow — share buttons v pozadí
  ostávajú renderované, fokus sa po zatvorení dialógu vráti na
  ResultsView (nie do share input).
- **AC-6**: Mobile: na <640px sa použije `<Sheet>` variant z E4.2
  (full-screen).
- **AC-7**: Cleanup: timer sa zruší na unmount (žiadny memory
  leak v hot-reload deve).
- **AC-8**: Tracking event `data_trap.shown`
  `{ source: "auto_timer" | "manual_button" }` cez `track()`.

## Subtasks

### Implementation
- [ ] V `ResultsView.tsx` pridať state `trapOpen: boolean`,
  `trapSeen: boolean` (z localStorage).
- [ ] `useEffect` na mount:
  - prečítať `iiq_trap_seen`
  - ak nie je `seen` → `setTimeout(() => setTrapOpen(true), 5000)`
  - cleanup: `clearTimeout`
- [ ] Pridať tlačidlo "Vyskúšaj si pasce na osobné údaje" v
  `ResultsView` body (pod share section), klik → `setTrapOpen(true)`.
- [ ] Render `<TrapDialog open={trapOpen} onOpenChange={...}
  onAcknowledged={() => localStorage.setItem("iiq_trap_seen","1")} />`.
  - `onAcknowledged` callback fired iba pri klike na "Som naivný"
    CTA z E4.2.
- [ ] Tracking call pre `data_trap.shown` v `useEffect` keď
  `trapOpen` flipne na true (rozlíšiť source cez druhý state).

### Tests
- [ ] **Timer test** (`tests/components/ResultsView.test.tsx`):
  - render ResultsView (no `iiq_trap_seen`)
  - `vi.useFakeTimers()`, advance 5000ms
  - assert `<TrapDialog>` v DOM s `open=true`
- [ ] **Skip-on-seen test**: pre-set
  `localStorage.setItem("iiq_trap_seen","1")` → render → advance
  5000ms → assert dialog NIE je open. Manual button ho otvorí.
- [ ] **Acknowledge flag test**: render, advance timer, otvorený
  dialog, klik "Som naivný" → assert
  `localStorage.getItem("iiq_trap_seen") === "1"`.
- [ ] **Pasive close NO flag test**: render, advance timer,
  otvorený dialog, ESC press → assert
  `localStorage.getItem("iiq_trap_seen")` ostáva null.
- [ ] **Cleanup test**: render → unmount pred 5000ms → no error,
  no late `setTrapOpen` call (use `act()` warning watcher).
- [ ] **Share regression test**: render, otvorený dialog,
  zatvoriť → assert share buttons sú stále funkčné (klik na
  "Copy link" → clipboard volaná).

## Riziká

| Riziko | Mitigácia |
|---|---|
| 5s timer je zlý moment (príliš skoro/neskoro) | Po deploy zmerať bounce/scroll engagement, prípadne nastaviť na 3s alebo 8s |
| Memory leak ak unmount pred 5s | Cleanup test (AC-7) |
| Používateľ je naštvaný že popup vyskočil počas čítania | "Som naivný" je single-click dismiss; budúca iterácia: scroll-position aware (otvor popup iba keď scroll = max) |

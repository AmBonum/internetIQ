# AH-11.4 — Public respondent flow Supabase wiring (/t/$shareId)

**Epic:** [AH-11 — Mock → Supabase wiring + production-ready verification](../PLAN-2026-05-17-admin-hub-integration.md)
**Effort:** `L` (≤ 1 week)
**Priority:** `P1` (must)
**Status:** Backlog

## Goal

Wire the public `/t/$shareId` respondent flow to Supabase via
`supabaseAdmin` with explicit safe-column projection, activate the CF
Pages rate-limit middleware, and confirm anon role has no direct write
path to `sessions` or `session_answers`.

## Acceptance criteria

- All ported files have `npx eslint --fix` applied; `npm run lint` is 0/0.
- `data-testid` values preserved from AH-8; no new test-ids added.
- i18n keys unchanged; no new namespaces.
- Server fn at `src/lib/respondent/take-test.functions.ts` returns only
  `id, title, description, intake_fields, gdpr_purpose, status` (PLAN
  "Public Respondent Flow Safety"). Never returns `owner_id`,
  `password_hash`, or `segmentation`.
- Anon role has zero `INSERT` policy on `sessions` or `session_answers`;
  all writes go through `createServerFn` validating `share_id` exists and
  `status = 'published'` before calling `supabaseAdmin`.
- CF Pages function middleware in `functions/_middleware.ts` enforces
  10 submissions / 5 min / IP; an 11th request inside the window returns
  HTTP 429.
- `forbid_session_score_changes` trigger (AH-1) blocks score mutation
  after `completed`; verified by integration test.
- Incognito smoke completes the full respondent flow end-to-end against
  the preview Supabase DB with zero auth cookies present.
- CONSENT_VERSION unchanged (1.4.0 from AH-7.1).

## Implementation

# E12.1 — Schema rozšírenie pre education mode

**Epic:** [E12 — Education mode](../PLAN-2026-04-26-custom-tests-sponsorship.md#epic-12--education-mode-autori-zbierajú-výsledky)
**Effort:** `S` (≤ ½ dňa)
**Priority:** `P2` (should)
**Status:** ✅ Done (2026-05-01)

## Kontext

Education mode je opt-in flag na test_sets. Keď je zapnutý, respondent
sa identifikuje menom + e-mailom a autor pristupuje k výsledkom cez
heslo. Schéma musí podporovať bezpečnú autentifikáciu (bcrypt) a
GDPR-friendly retenciu PII.

## User story

> Ako učiteľ vytvárajúci test pre triedu, **chcem** uložiť svoj test
> s heslom a možnosťou zbierať mená respondentov, **aby** som po teste
> vedel ohodnotiť konkrétnych študentov.

## Acceptance criteria

- **AC-1**: Migrácia `supabase/migrations/{timestamp}_edu_mode.sql`:
  ```sql
  ALTER TABLE public.test_sets
    ADD COLUMN collects_responses BOOLEAN NOT NULL DEFAULT false,
    ADD COLUMN author_password_hash TEXT,           -- bcrypt $2a$10$...
    ADD CONSTRAINT test_sets_edu_pwd_required CHECK (
      collects_responses = false OR author_password_hash IS NOT NULL
    );

  ALTER TABLE public.attempts
    ADD COLUMN respondent_name TEXT,
    ADD COLUMN respondent_email TEXT,
    ADD CONSTRAINT attempts_respondent_email_format CHECK (
      respondent_email IS NULL OR respondent_email ~* '^[^@\s]+@[^@\s]+\.[^@\s]+$'
    ),
    ADD CONSTRAINT attempts_edu_pii_pair CHECK (
      (respondent_name IS NULL AND respondent_email IS NULL) OR
      (respondent_name IS NOT NULL AND respondent_email IS NOT NULL)
    );

  CREATE INDEX attempts_test_set_id_created_at_idx
    ON public.attempts (test_set_id, created_at DESC)
    WHERE test_set_id IS NOT NULL;
  ```
- **AC-2**: RLS update — anon stále môže INSERT (s validáciou cez constraint),
  ale anon **NEMÔŽE SELECT** riadky kde `respondent_name IS NOT NULL` (PII protection).
  Public view `attempts_anon` filtruje out `respondent_*` stĺpce → anon SELECT
  cez tento view, nie priamo z `attempts`.
- **AC-3**: Server-side validation v INSERT path:
  - ak `test_set.collects_responses = true` → `respondent_name + respondent_email` musia byť NOT NULL
  - ak `test_set.collects_responses = false` → musia byť NULL
  Validácia v Supabase RPC funkcii (nie len constraint) lebo treba cross-table check.
- **AC-4**: Author password verification RPC `verify_test_set_password(set_id, password)` →
  vráti boolean. SECURITY DEFINER, bcrypt comparison. Brute-force protekcia
  na rate-limit úrovni v E12.4 (5 pokusov / 15 min).
- **AC-5**: Retention extension — `purge_unused_test_sets` zachováva existing
  12-mes pravidlo. Pre edu mode sa pridáva `purge_expired_respondent_pii()`
  ktorá nullify-uje `respondent_name + respondent_email` v `attempts` kde
  `created_at < now() - interval '12 months'` — skóre + answers zostávajú.
- **AC-6**: Migrácia tiež do `DEPLOY_SETUP.sql`. Supabase types regenerated.

## Riziká

| Riziko | Mitigácia |
|---|---|
| bcrypt v Supabase Postgres extension nie je available | pgcrypto má `crypt()` + `gen_salt('bf')` — bf = blowfish ekvivalent bcrypt |
| Email regex príliš permisívny | RFC 5322 plný regex je overhead; akceptujeme jednoduchý formát + necháme overiť MX cez bounce |
| Retention cron zmaže PII pred semestrom | 12 mesiacov je dosť pre 1 školský rok; auto can change v migrácii ak treba |

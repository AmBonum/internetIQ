## Kontext

Implementačná story pre samotný popup. Najkritickejší aspekt:
**žiadny vstup používateľa nesmie opustiť prehliadač**. Tento
princíp je súčasťou právnej obrany (privacy.tsx ho explicitne
deklaruje) a kritického UX bonu — používateľ MUSÍ uveriť že
sme ho neoklamali.

## User story

> Ako návštevník po teste, **keď** vyplním pole rodným číslom,
> **chcem** okamžite (pred submitom) vidieť výrazné varovanie
> v rovnakej karte, **aby** som si zapamätal/a že tento údaj
> sa nezadáva do náhodných formulárov.

## Acceptance criteria

- **AC-1**: `src/components/quiz/TrapDialog.tsx` — controlled dialog
  (`open`, `onOpenChange` props) s ~6 input polami zo schváleného
  zoznamu (E4.1).
- **AC-2**: **Žiadny `onSubmit`. Žiadny `fetch` / `axios` / supabase
  call. Žiadny `localStorage.setItem` na vstupné dáta** (iba
  `iiq_trap_seen` flag, žiadne hodnoty polí).
- **AC-3**: Per-pole real-time pattern matcher: zmena `<input>`
  → `useEffect` na detect → ak match → render warning box pod
  poľom (animated slide-down).
- **AC-4**: Warning box má 3 vizuálne intensity podľa
  `citlivosť` z E4.1:
  - `kritická`: red-600 border + shake animation
  - `vysoká`: orange-500 border, no shake
  - `stredná`: yellow-500 border, subtle
- **AC-5**: Footer dialógu má 1 CTA **"Som naivný, nech to už
  nikdy neurobím"** (ironicky, zatvára dialog cez `onOpenChange(false)`).
  Žiadne "submit" / "uložiť" / "pokračuj".
- **AC-6**: Druhé tlačidlo "Zatvoriť" v hornom rohu dialógu
  (Radix DialogClose) — pre používateľov čo nechcú interagovať.
- **AC-7**: A11y:
  - `role="dialog"`, `aria-labelledby`, `aria-describedby`
  - Focus trap počas open
  - ESC zatvára
  - Focus return na element ktorý dialog otvoril
- **AC-8**: Mobile UX: na <640px (sm breakpoint) → render ako
  full-screen `<Sheet>` (Radix sheet), nie centrovaný dialog
  (lepšia kontrola, menej claustrofóbie).
- **AC-9**: Tracking event `data_trap.field_warned` cez `track()`
  helper z `src/lib/tracking.ts`:
  ```ts
  track(consent, {
    name: "data_trap.field_warned",
    properties: { field: "birth_number" },
    category: "analytics"
  });
  ```
  No-op bez analytics consent.
- **AC-10**: Pattern matchers v separátnom module
  `src/lib/data-trap/matchers.ts` — exportujú čistú funkciu
  `(value: string, fieldId: string) => boolean`.

  ```ts
  }
  ```

## Riziká

| Riziko | Mitigácia |
|---|---|
| False positive — IČO interpretované ako card | Luhn checksum + 3+3 negative samples test |
| Test "no fetch called" je flaky kvôli HMR / dev timing | Mock all I/O; test beží v Node + jsdom, nie real browser |
| Používateľ devtools-om vidí matchery a obíde popup | Akceptovateľné — appka nemá brániť pokročilému atakeru, je edukačná |
| Performance — debouncing nie je implementovaný | Matcher beží len pri `onChange`, ktoré samo throttluje keystroky; debounce skip |

# E10.2 — `sponsors` + `donations` + `subscriptions` migration + RLS

**Epic:** [E10 — Sponsorship infrastructure](../PLAN-2026-04-26-custom-tests-sponsorship.md#epic-10--sponsorship-infrastructure)
**Effort:** `S` (≤ ½ dňa)
**Priority:** `P1` (must)
**Status:** ⛔ Blocked by E10.1

## Kontext

DB schéma pre sponzorstvo. 3 tabuľky + verejný view pre `/sponzori`
zoznam. RLS striktne anon-only SELECT na public_sponsors view.

## User story

> Ako vývojár implementujúci sponzor flow, **chcem** stable DB schému
> s RLS, **aby** webhook vedel ukladať bez data leakov, a frontend
> vedel pretvárať verejný zoznam darcov.

## Acceptance criteria

- **AC-1**: Migrácia `supabase/migrations/{timestamp}_sponsors.sql`:
  ```sql
  CREATE TABLE public.sponsors (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    stripe_customer_id TEXT UNIQUE NOT NULL,
    display_name TEXT,            -- NULL = anonymous
    created_at TIMESTAMPTZ DEFAULT now(),
    total_eur NUMERIC(10,2) DEFAULT 0
  );

  CREATE TABLE public.donations (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    sponsor_id UUID REFERENCES sponsors(id) ON DELETE CASCADE,
    stripe_payment_intent_id TEXT UNIQUE,
    amount_eur NUMERIC(10,2) NOT NULL,
    currency TEXT NOT NULL DEFAULT 'EUR',
    kind TEXT NOT NULL CHECK (kind IN ('oneoff','subscription_invoice')),
    invoice_pdf_url TEXT,
    created_at TIMESTAMPTZ DEFAULT now()
  );

  CREATE TABLE public.subscriptions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    sponsor_id UUID REFERENCES sponsors(id) ON DELETE CASCADE,
    stripe_subscription_id TEXT UNIQUE,
    status TEXT NOT NULL,
    monthly_eur NUMERIC(10,2) NOT NULL,
    started_at TIMESTAMPTZ DEFAULT now(),
    cancelled_at TIMESTAMPTZ
  );

  CREATE VIEW public.public_sponsors AS
  SELECT id, display_name, created_at, total_eur
  FROM sponsors WHERE display_name IS NOT NULL;
  ```
- **AC-2**: RLS:
  - `sponsors`, `donations`, `subscriptions`: RLS enabled, **žiadny anon SELECT/INSERT/UPDATE/DELETE**. Webhook používa service role key (na serveri, nie v browseri).
  - `public_sponsors` view: anon SELECT povolený.
- **AC-3**: Indexy: `sponsors.stripe_customer_id`, `donations.sponsor_id`, `subscriptions.sponsor_id`, `donations.created_at DESC`.
- **AC-4**: Trigger `update_sponsor_total` — pri INSERT do `donations` aktualizuje `sponsors.total_eur`.
- **AC-5**: Migrácia tiež do `DEPLOY_SETUP.sql`.
- **AC-6**: Supabase types regenerated.

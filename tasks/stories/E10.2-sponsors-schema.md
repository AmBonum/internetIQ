## User story

## Acceptance criteria

- **AC-1**: Migrácia `supabase/migrations/{timestamp}_sponsors.sql`:
  ```sql
  CREATE TABLE public.sponsors (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    stripe_customer_id TEXT UNIQUE NOT NULL,
    display_name TEXT,            -- NULL = anonymous (default OFF)
    display_link TEXT,            -- voliteľný https:// link; NULL = none
    display_message TEXT,         -- voliteľná krátka správa, max 80 znakov
    show_in_footer BOOLEAN DEFAULT false,
    created_at TIMESTAMPTZ DEFAULT now(),
    total_eur NUMERIC(10,2) DEFAULT 0,
    CONSTRAINT sponsors_display_message_len CHECK (
      display_message IS NULL OR length(display_message) <= 80
    ),
    CONSTRAINT sponsors_display_link_https CHECK (
      display_link IS NULL OR display_link LIKE 'https://%'
    )
  );

  CREATE TABLE public.donations (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    -- ON DELETE RESTRICT: zákonná povinnosť uchovať účtovné záznamy 10
    -- rokov per zákon č. 431/2002 Z.z. — sponsor sa nedá DELETE pokiaľ má
    -- donations. GDPR Art. 17 sa rieši anonymizáciou (display_name → NULL),
    -- nie cascade delete (E11.5 runbook).
    sponsor_id UUID REFERENCES sponsors(id) ON DELETE RESTRICT,
    stripe_payment_intent_id TEXT UNIQUE,
    amount_eur NUMERIC(10,2) NOT NULL,           -- môže byť negatívna pri refunde (audit trail)
    currency TEXT NOT NULL DEFAULT 'EUR',
    kind TEXT NOT NULL CHECK (kind IN ('oneoff','subscription_invoice','refund')),
    refund_of_donation_id UUID REFERENCES donations(id),  -- self-FK pre refund linkage
    invoice_pdf_url TEXT,
    created_at TIMESTAMPTZ DEFAULT now()
  );

  CREATE TABLE public.subscriptions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    sponsor_id UUID REFERENCES sponsors(id) ON DELETE RESTRICT,
    stripe_subscription_id TEXT UNIQUE,
    status TEXT NOT NULL,
    monthly_eur NUMERIC(10,2) NOT NULL,
    started_at TIMESTAMPTZ DEFAULT now(),
    cancelled_at TIMESTAMPTZ
  );

  CREATE VIEW public.public_sponsors AS
  SELECT id, display_name, display_link, display_message, created_at
  FROM sponsors WHERE display_name IS NOT NULL;

  -- Footer-tier sponsors (gated by cumulative spend; consent-gated separately
  -- via show_in_footer flag).
  CREATE VIEW public.footer_sponsors AS
  SELECT s.id, s.display_name, s.display_link
  FROM sponsors s
  LEFT JOIN subscriptions sub ON sub.sponsor_id = s.id AND sub.status = 'active'
  WHERE s.show_in_footer = true
    AND s.display_name IS NOT NULL
    AND (s.total_eur >= 50 OR sub.monthly_eur >= 25);
  ```
  Pozn: `total_eur` je v `public_sponsors` view zámerne vynechané — ne-exponujeme ho ako PII. Len `footer_sponsors` view ho používa interne v WHERE.
- **AC-2**: RLS:
  - `sponsors`, `donations`, `subscriptions`: RLS enabled, **žiadny anon SELECT/INSERT/UPDATE/DELETE**. Webhook používa service role key (na serveri, nie v browseri).
  - `public_sponsors` view: anon SELECT povolený.
- **AC-3**: Indexy: `sponsors.stripe_customer_id`, `donations.sponsor_id`, `subscriptions.sponsor_id`, `donations.created_at DESC`.
- **AC-4**: Trigger `update_sponsor_total` — pri INSERT do `donations` aktualizuje `sponsors.total_eur += NEW.amount_eur`. Pri `kind='refund'` je `amount_eur` negatívne, takže trigger automaticky decrement-uje. Pre footer_sponsors view to znamená že sponsor po refunde môže prepadnúť pod 50 € prah a zmizne z footra.
- **AC-7**: Anonymization SOP (GDPR Art. 17 vs. zákon o účtovníctve): `UPDATE sponsors SET display_name=NULL, display_link=NULL, display_message=NULL, show_in_footer=false WHERE id=$1` — donations zostávajú s sponsor_id linkom, ale public-facing PII zmizne. Documented v E11.5 runbook.
- **AC-5**: Migrácia tiež do `DEPLOY_SETUP.sql`.
- **AC-6**: Supabase types regenerated.

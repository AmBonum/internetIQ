# E10.3 — CF Pages Function: Stripe webhook handler

**Epic:** [E10 — Sponsorship infrastructure](../PLAN-2026-04-26-custom-tests-sponsorship.md#epic-10--sponsorship-infrastructure)
**Effort:** `M` (≤ 2 dni)
**Priority:** `P1` (must)
**Status:** ⛔ Blocked by E10.2

## Kontext

Stripe webhook je **single trust boundary** medzi platobným procesorom a
našou DB. Verifikuje signature, pretvára Stripe events na DB writes. Beží
ako Cloudflare Pages Function (`functions/api/stripe-webhook.ts`), využíva
service role key pre Supabase.

## User story

> Ako vývojár, **chcem** robust webhook ktorý spracuje Stripe events
> idempotentne a bezpečne, **aby** žiadna platba nebola stratená alebo
> dvakrát zaznamenaná.

## Acceptance criteria

- **AC-1**: Súbor `functions/api/stripe-webhook.ts` exporting POST handler:
  - Verifikuje `Stripe-Signature` header podľa `STRIPE_WEBHOOK_SECRET`.
  - Ak verifikácia zlyhá → return 400.
  - Parses event a deleguje na handler podľa `event.type`.
- **AC-2**: Spracované event types:
  - `checkout.session.completed` (oneoff donation) → upsert sponsor + insert donation
  - `invoice.paid` (subscription invoice) → insert donation s kind `subscription_invoice`
  - `customer.subscription.created` → insert subscription
  - `customer.subscription.updated` → update subscription status
  - `customer.subscription.deleted` → mark subscription cancelled
- **AC-3**: Idempotencia: pred INSERT donation skontroluj `stripe_payment_intent_id` UNIQUE constraint, conflict → return 200 silently (Stripe retry-uje pri non-2xx).
- **AC-4**: AML guard: ak `amount_eur > 500` → return 400 (single-tx limit). Stripe automaticky vráti platbu.
- **AC-5**: Logging: every event logged do CF Pages Function logs (no PII, len event id + type + amount).
- **AC-6**: Env vars: `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`, `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`.
- **AC-7**: Rate limit: ak 10+ requestov / minútu z 1 IP, return 429 (Cloudflare Turnstile alebo CF native rate limiting).
- **AC-8**: TypeScript strict, `any`-free.

## Riziká

| Riziko | Mitigácia |
|---|---|
| CF Pages Function timeout (10s) pri pomalej DB | Stripe retry-uje, idempotency to vykryje |
| Service role key leak | Iba CF env var, nikdy v repo, nikdy v client kóde |
| Rate-limit obíde útočník cez CDN | Cloudflare native rate limit po IP funguje na edge level |

# AH-10.1 — Admin shell + AdminSidebar + role-gate middleware

**Epic:** [AH-10 — Admin panel role-gating + shell](../PLAN-2026-05-17-admin-hub-integration.md#epic-map)
**Effort:** `M`
**Priority:** `P1`
**Status:** 🟡 Ready
**Depends on:** AH-1.1, AH-3.1
**Source in admin-hub:** `routes/admin.tsx`, `components/admin/AdminSidebar.tsx`

## Goal
Port the `/admin/*` layout shell and AdminSidebar, and install the server-side role gate. The gate combines `requireSupabaseAuth` (session present) with `requireRole('admin')` (the session's user has the admin role per `has_role()` defined in AH-1.1). Unauthenticated → 401; authenticated non-admin → 403.

## Acceptance criteria
- `src/routes/admin.tsx` layout mounts AdminSidebar and applies `requireSupabaseAuth` followed by `requireRole('admin')`.
- Unauthenticated request to `/admin` or any child returns 401 redirect to `/prihlasit`; authenticated non-admin returns 403.
- `src/integrations/supabase/role-middleware.ts` exports `requireRole(role)` — a `createServerFn` middleware that calls `has_role(user_id, role)` via the anon client.
- All ported files run through `npx eslint --fix`.
- Data-testids: `admin-shell-sidebar`, `admin-shell-sidebar-link-dashboard`, `admin-shell-sidebar-link-users`, `admin-shell-sidebar-link-questions`, `admin-shell-sidebar-link-tests`, `admin-shell-sidebar-link-support`, `admin-shell-sidebar-link-settings`, `admin-shell-sidebar-collapse-toggle`.
- Slovak strings extracted to `src/i18n/locales/sk/admin.json` and registered in `src/i18n/resources.ts` under the `admin` namespace.
- Reads `user_roles` via `has_role()` only; no other Supabase calls in this story (mock store powers child routes).

# E11.5 — Refund SOP + AML threshold playbook

**Epic:** [E11 — Sponsorship UI + invoicing](../PLAN-2026-04-26-custom-tests-sponsorship.md#epic-11--sponsorship-ui--invoicing)
**Effort:** `S` (≤ ½ dňa)
**Priority:** `P3` (nice)
**Status:** ✅ Done — runbook at [`tasks/E11-sponsorship-runbook.md`](../E11-sponsorship-runbook.md) covers refund flow (full/partial/decline), subscription cancel follow-up, DB cleanup edge cases, AML cap response, GDPR Art. 17 anonymization SQL (constrained by zákon č. 431/2002 10-year retention), Stripe dispute counter-evidence, e-mail delivery failures + 4 SK-localized e-mail reply templates.

## Kontext

Edge cases nie sú v MVP UI, ale ak sa stane refund alebo prekročenie
AML, treba mať jasný runbook (aj keď nie automatický).

## User story

> Ako product owner / účtovník, **chcem** krátky dokument ako riešiť
> refund požiadavku alebo AML alert, **aby** som vedel rýchlo zareagovať
> bez improvizácie.

## Acceptance criteria

- **AC-1**: `tasks/E11-sponsorship-runbook.md` so sekciami:
  1. **Refund flow** — kedy je full refund OK (do 14 dní pri ne-poskytnutej službe), kedy partial / no. Webhook automaticky decrement-uje `total_eur` cez `charge.refunded` event (E10.3 AC-2). Manuálny krok: skontrolovať či sponzor prepadol pod footer threshold (≥ 50 € one-off alebo ≥ 25 € monthly) → ak áno, jeho footer mention zmizne automaticky cez `footer_sponsors` view filter.
  2. **Stripe dashboard kroky** — refund button, refund reason, tax adjustment, idempotency note.
  3. **Subscription cancellation manuálny follow-up** — keď sponsor zruší cez Customer Portal, webhook označí `subscriptions.cancelled_at`; konateľ dostane email cez E11.8 infra; manuálne kroky:
     - Skontrolovať, či sponzor stále kvalifikuje pre footer (cez `total_eur` cumulatív z one-off donations)
     - Žiadne ďalšie tech kroky (žiadny Discord ani iné externe služby)
  4. **DB cleanup** — manuálny SQL update `donations.invoice_pdf_url = NULL` ak treba; sponsor.total_eur sa automaticky aktualizuje cez trigger.
  5. **AML alert** — čo robiť ak AML threshold prekročený (webhook vráti 400, transakcia neprešla; ak by sa prešmyklo, manuálny refund + report do FIU SR podľa zákona č. 297/2008 Z.z.).
  6. **Sponzor žiada vymazanie** (GDPR Art. 17) — kedy to nemôžeme spraviť (10 rokov uchovanie účtovných dokladov per zákon č. 431/2002 Z.z.), čo môžeme (`UPDATE sponsors SET display_name=NULL, display_link=NULL, display_message=NULL, show_in_footer=false`). Donations zostávajú s sponsor_id linkom pre audit trail. Email sponzorovi s vysvetlením 10-rocnej retencie.
  7. **Stripe dispute (chargeback)** — kroky, evidence to submit cez Stripe dashboard.
  8. **Email delivery failure** — ak Resend vráti error, manuálny re-send cez Stripe dashboard pre receipty alebo direct email z `subenai.podpora@gmail.com` pre magic link.
- **AC-2**: Šablóny emailových odpovedí v SK pre každý scenár.
- **AC-3**: Step-by-step z account-ovho pohľadu (pre netechnického).

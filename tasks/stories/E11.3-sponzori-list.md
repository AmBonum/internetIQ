# E11.3 — `/sponzori` public donor list (consent-gated)

**Epic:** [E11 — Sponsorship UI + invoicing](../PLAN-2026-04-26-custom-tests-sponsorship.md#epic-11--sponsorship-ui--invoicing)
**Effort:** `S` (≤ ½ dňa)
**Priority:** `P2` (should)
**Status:** ⛔ Blocked by E11.1

## Kontext

Verejná stránka so zoznamom sponzorov, ktorí pri checkout-e zaškrtli
„Zobraziť moje meno". Žiadne hodnoty platieb sa neukazujú (anonymizované),
len mená a dátumy.

## User story

> Ako sponzor, ktorý dal súhlas, **chcem** sa vidieť na verejnom zozname,
> **aby** som mohol byť hrdý na podporu projektu a inšpirovať ostatných.

## Acceptance criteria

- **AC-1**: Route `/sponzori` s `<title>Naši sponzori — Internet IQ Test</title>`.
- **AC-2**: Hero: „Vďaka týmto ľuďom funguje Internet IQ Test."
- **AC-3**: Grid kartičiek z `public_sponsors` view (cez anon RLS):
  - `display_name`
  - `created_at` (mesiac + rok)
  - **NIE** total_eur, ani konkrétna čiastka, ani počet donations
  - Voliteľný „🎉 Mesačný sponzor" badge ak má aktívnu subscription (cez ďalší public view)
- **AC-4**: Empty state: „Buď prvý!"
- **AC-5**: Sortovanie: najnovší prvý.
- **AC-6**: SEO: indexable, `noindex` *NIE*; OG tags štandardné.
- **AC-7**: Page footer: malý text — „Zoznam je dobrovoľný; mnohí naši sponzori si zvolili anonymitu."

## Subtasks

### Implementation
- [ ] Route file s loader na `public_sponsors`.
- [ ] `<SponsorCard>` komponent.
- [ ] Empty state.

### Tests
- [ ] Render test s mock sponzorov.
- [ ] Anon SQL test: `SELECT FROM public_sponsors WHERE display_name IS NULL` → 0 rows (view-level filter).

### Documentation
- [ ] Update privacy stránku o tom, ako odvolať súhlas (žiadosť na email — neskôr možno self-service).

### Code review (fresh context)
